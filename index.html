2. Formulating and Solving the Mathematics

2.1 Understanding what is Travelling Salesman Problem (TSP)

Assuming a salesman has to visit n cities/places. He wishes to start from a particular city, visit each city once and return to his starting point. His objective is to select the sequence in which the cities are visited in such a way that his total travelling cost or time is minimized.

Conditions: No city is to be visited twice before all of the cities are visited atleast once. In short, The Traveling Salesman Problem is a well-known optimization problem that seeks to find the shortest possible route that visits a set of locations exactly once and returns to the starting location.

2.3 Mathematical Formula/Algorithm

There is a mathematical formula/algorithm associated with this.



→g(i, S) = min[w(i, j) + g(j, {S - j})]

where j ∈ S, i ≠ j and w(i, j) is distance between i and j.
i = Starting vertex/city (here, A).
S = Set of vertices/cities that salesman will visit exactly once (here, B, C and D).
This formula follows Recursion method to find the result.
